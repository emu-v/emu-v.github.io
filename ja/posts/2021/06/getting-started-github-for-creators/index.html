【黒い画面不要】もうUnityプロジェクトが壊れても泣かない、時を戻せる魔法「GitHub」入門【無料です】

VRChatでアバターやワールドを作っていると「いつの間にかUnityプロジェクト壊れてた 🥺 」みたいな悲しいことが起こります。

でも「GitHub（ギットハブ）」を使えば壊れる前まで時を戻せるようになります。

この記事ではそんな方法を、できる限り簡単に、どうしても必要な部分だけご紹介します。
WindowsでもMacでも同じ手順 で使えますし、GitHubの無償機能だけを使いますので 追加費用もかかりません。

アナログ人間を自認するぼくのお友達を思い浮かべながら書いたので、プログラマさんが使ってる 黒い画面や、呪文詠唱より難しそうな謎のコマンドは登場しません。

その代わりに、ガチプログラマの方にとっては突っ込みたくなる表現や手順があると思います。
それはぜひツイッターでコメントお願いします！

GitHubってなに？

GitHubはUnityプロジェクトのようなフォルダとその内容、あるいはBlenderやPhotoshopで作ったファイルを世代管理する「バージョン管理」のクラウドサービスです。

バージョン管理とは、例えばアバターさんの色改変に次のような「バージョン」を付けて、そのバージョンを戻したり進めたりできるようにすることです。

  • 春に作った花柄服着たこんちゃん→バージョン 2021.5.1
  • 七夕に合わせた浴衣こんちゃん→バージョン 2021.7.1
  • ハロウィンこんちゃん→バージョン 2021.10.1

バージョンはよくver.1.2.3 みたいな付け方をしますが、細かい話になるのでこの記事では省略します。

GitHubは、この付けたバージョン毎に、Unityプロジェクトなどのファイル一式を預かってくれるサービスです。

この記事で紹介する手順でGitHubのアカウントを作って、ファイル一式を送ると、世界のどこかにあるサーバーに、ぼくやあなたの作ったファイルが預けられます。

GitHubでできること

GitHubでバージョン管理すると次のようなことができるようになります。

  1. Unityプロジェクトや他のファイルを世代管理して、壊れたり失敗しても過去の状態に戻すことができます。
  2. 複数のPCでUnityプロジェクトや他のファイルを交互に編集したり、新しいPCにこれらのファイルを復元できます。
  3. （この記事では扱いません）例えばあるテクスチャを赤と青どちらを基調にしようか迷ったとき、分岐して２つのバージョンを切り替えながら作業を進めることができます。
  4. （この記事では扱いません）あなたが指定した別の人とファイルを共有して共同で作業できます。

この記事の範囲

冒頭でご説明した通り、この記事の目的は必要最小限の学習と操作でUnityプロジェクトや他のファイルをバージョン管理し、守ることです。

そのためGitHubの多くの高度な機能はご説明せず、次のごく一部の機能と手順だけをご紹介します。

でも大丈夫です。
これらの機能だけを使い始めてもGitHubの恩恵を受けることができます。

無料でGitHubアカウントを取得する

まずはGitHubのアカウントを登録してみましょう！
今回ご紹介する内容はすべて無料です。

GitHubアカウントを作るために用意するものは

  • Emailアドレス
  • 使いたいパスワード
  • 使いたいアカウント名

の3つです。

ブラウザのアドレス欄に「github.com」と打ち込むと、2021年6月現在ではこんな画面が表示されます。
右上のSign Upボタンをクリックしましょう。

Email、パスワード、アカウント名をそれぞれ入力しましょう。
その他、メルマガの登録や「あなたは人間ですか？」のチェックがあります。

全ての項目に答えると次のような画面に移動し、登録したメールアドレス宛に確認メールが届きます。

メール内のボタンをクリックして登録完了です。

GitHubのアカウント作成が完了すると次のような画面が表示されます。

Web上でGitHubを試す

最初にブラウザだけで「バージョン管理」を試します。
テキストファイルを作って、書き換えて、過去に戻せる様子を体験しましょう！

リポジトリとは？

まずは管理対象となる「リポジトリ」を作りましょう。
リポジトリは、WindowsやMacのドライブやフォルダのようなものです。
Unityプロジェクトなどバージョン管理したい対象をこのリポジトリの中に入れます。

順にやっていきましょう。

GitHubにサインインした状態で、右上の＋ボタンをクリックするといくつかのメニューが表示されます。
その中から「New repository」をクリックします。

そうすると次のような画面が表示されます。順に入力または選択していきましょう。

「Repository name」はアルファベットでお好きな名前を入力しましょう。数字やハイフンも使えます。
ここでは「vrchat-avatars-sandbox」にします。

Public / Privateは「Private」を選択します。
「Public」にすると世界中の人にこれから作るUnityプロジェクトなどが見えてしまいます。
VRChatのアバターやワールドに関しては常にPrivateに設定しておけば間違いないでしょう。

「Add a README file」は説明ファイル作るかどうかの選択です。チェックを入れます。

「Add .gitignore」もUnityプロジェクトを管理するときは便利な設定です。チェックを入れ、画像のように「.gitignore template」の入力欄に「unity」と入力し表示されたものを選択します。

最後に「Create repository」ボタンをクリックしましょう。

以上でリポジトリが作られます。
線を引いたリポジトリ名の部分をクリックすると、いつでもこのリポジトリのトップ画面に戻れます。
この記事の中で何度かページを行き来するので、迷子になったときはリポジトリ名をクリックしてこのページに戻ってくると便利です。

まずはテキストファイルを「コミット」する

では早速「バージョン管理」を体験しましょう！
まずはこのままブラウザを使ってWeb上だけで試します。

テキストファイルを１つ作ります。
リポジトリのトップ画面にある「Add file」メニューから「Create new file」をクリックします。

次のような画面に移動するので丸で囲んだ箇所に作るファイル名を入力します。
ファイル名はアルファベット、数字、ハイフン、アンダースコアだけにしておくと安心です。
また今回はファイル名の最後を「.txt」にします。

ここでは「memo.txt」としました。

「Edit new file」の欄に何か１行書き込んでみましょう。

次に「Commit new file」ボタンをクリックします。
この操作をコミット（Commit）と呼びます。

リポジトリのトップ画面を見ると今作った「memo.txt」が追加されています。
この「memo.txt」をクリックしてみましょう。

次のような画面に移動し、先ほど書いた内容が表示されます。
これが「コミット」で、追加や変更した内容をGitHubに預ける操作です。

リポジトリのトップ画面を表示し、今度は丸で囲んだ「（数字）commits」と表示されている部分をクリックしてください。

次のような画面に移動します。
これは先ほどのような操作でコミットした内容の一覧です。

ややこしいのですが、コミットした内容の１つ１つも「コミット（Commit）」と呼ばれます。
操作も内容も「コミット（Commit）」というわけです。

内容の方のコミットはバージョンの元あるいは材料になります。
この記事では扱いませんが、この「コミット」に「ver.1.2.3」のような「タグ」を割り当てて「バージョン」として扱います。

2種類のコミットが混在するのは混乱の元なので、内容の方のコミットをこの記事では「コミット（バージョン）」と書きます。

このコミット（バージョン）を開くと、追加した行らしきものが「＋」記号と共に表示されています。
これは「＋記号で始まる行が追加された」という意味です。

さらにテキストファイルを編集して「コミット」する

続けて同じテキストファイルに2行目を書き込んでみましょう。
リポジトリのトップ画面からテキストファイルをクリックします。

今度は右下の鉛筆のようなマークをクリックします。

そうすると先ほどと同じようなテキストを編集できる画面に移動するので、2行目を追加します。

そして「Commit changes」ボタンをクリックします。

コミットが成功するとこのような画面に移動します。

丸で囲んだ部分をクリックしてリポジトリのトップ画面に移動します。

「（数字）commits」と表示されている部分が増えていることが確認できます。
これはコミット（バージョン）が追加されたことを表しています。

ブラウザで次のような画面が開きます。

このアプリケーションをインストールすればWindowsでもMacでも同じ方法が使えますし、無料ですし、何より黒い画面に謎のコマンドを打ち込まずに済みます。

早速インストールしましょう。

ダウンロードボタンをクリックするとお使いのOSに合ったインストーラがダウンロードされます。

次のような画面が表示されるので「Sign in to GitHub.com」ボタンをクリックします。

ブラウザでの認証に成功すると「Open GitHubDesktop?」と尋ねられますので「Open GitHubDesktop」をクリックします。

GitHub Desktopアプリの右側に先ほど作ったリポジトリが表示されているはずなのでクリックします。

「Clone （あなたのアカウント名）/（あなたのリポジトリ名）」と書かれたボタンをクリックします。

次のようにどこにクローンするか設定する画面が表示されます。

Commit & Pushとは？

では次に、ローカルでUnityプロジェクトを作り、クラウド上に預けてみましょう！
この操作はコミット（Commit）とプッシュ（Push）の2段階に分かれますので順にやってみましょう。

サンプルのUnityプロジェクトを作る

まずは見慣れたUnity Hubの画面でUnityバージョンを指定してUnityプロジェクトを作ります。

この記事ではVRCSDKやアセットのインポートは省略します。
実際には何かをインポートしたり保存する度にこのコミットの手順を行なっておくことをお勧めします。
細かくコミットすることで、何か問題が発生したときにより細かくどの時点か指定して時を戻せるようになります。

Unityプロジェクトの保存先には、先ほどリポジトリをクローンした場所を指定します。

指定通りUnityプロジェクトがクローンしたリポジトリ内に作られます。

.gitignore ファイルを置く

次にコミットするのですが、Unityプロジェクトの場合はコミット前に一手間必要です。

GitHubのWeb上でリポジトリ作成時に作った「.gitignore（ドット ギット イグノア）」ファイルを使います。
このファイルを所定の場所に置くことで「コミットする必要がないファイル」を除外してくれます。

まず今作ったばかりのUnityプロジェクトを確認します。

右上のGitHub Desktopを見ると、線を引いた箇所に「1946 changed files」と書かれています。
これは、このままコミットするとGitHubに1946個のファイルが預けられることを示しています。

しかし実はこれらのファイルのほとんどはバージョン管理する必要がありません。
Unityが作業用に自動で作ったり消したりする一時的なファイルが多く含まれているためです。

そこで「.gitignore」ファイルをUnityプロジェクトのフォルダの中にコピーします。

そしてもう一度GitHub Desktopの画面を見ると「23 changed files」と表示が変わりました。
管理対象のファイルが1946から23まで大幅に減ったことを示しています。

「1946」や「23」といった数は違っても問題ありませんが、もし数が全く変わらない場合は操作を失敗していないか確認しましょう。
また、GitHub Desktopの表示が変わらない場合は、アプリ左上部の「Changes」「History」と書かれたタブを切り替えることで表示内容が更新されます。

このように「.gitignore」ファイルをUnityプロジェクトのフォルダに入れることで、バージョン管理不要なファイルを管理対象から外すことができました。

この「.gitignore」ファイルを置く操作は、Unityプロジェクトを追加した際に毎回必要です。

Commitする

これで作ったUnityプロジェクトをGitHubさんに預ける準備ができました。
ではコミットしましょう。

GitHub Desktopのこの部分にコメントを書きボタンを押すことでコミットできます。
このコメントは日本語や絵文字を使っても大丈夫です。
「Unityプロジェクト作成」「○○をインポート」など変更内容をコメントとして残しておくことで、時を戻したくなったときにわかりやすくなります。

また、コミットはあなたのローカルPCだけで完結しますのでWi-FiやLANに繋がっていなくてもできます。

Pushする

次に、コミットしたコミット（バージョン）をGitHubに送りましょう。
世界のどこかにあるクラウドサーバーに送りつけるイメージです。
この操作をプッシュ（Push）と呼びます。

操作はGitHub Desktopアプリの「Push origin」ボタンをクリックするだけです。

プッシュしてローカルPCからクラウドにコミット（バージョン）を預けておくことで、仮に手元のデータが消えてもクラウドから復元できるようになります。

逆に、プッシュされていないコミット（バージョン）はお手元から消えれば永久に失われてしまいますので気をつけてください。
慣れるまではコミットしたら必ずプッシュしておくと安心です。

さらにUnityプロジェクト編集しCommit & Pushする

練習としてUnityプロジェクトにCubeを置いてコミットしプッシュしてみましょう。

Unityプロジェクト特有の問題として、シーンが保存されているか、またビルド中などで一時的なカメラやオブジェクトが配置されていないか注意します。

Cubeを置いてシーンを保存するとGitHub Desktopアプリで変更箇所が表示されます。

1つのシーンにCubeを置いただけであれば変更はシーンファイル1つだけでしょう。
このように行った変更とGitHub Desktopアプリの表示を見比べることで、変更内容が想定通りかある程度確認できます。

GitHub Desktopアプリ上でコミットしプッシュします。

Revertとは？

ここでようやく時を戻す方法の登場です。
時を戻すとはつまり「現在のコミット（バージョン）を取り消し、過去のコミット（バージョン）を復元すること」です。

この操作をリバート（Revert）と呼びます。

先ほどCubeを置いたUnityプロジェクトを、Cubeを置く前に戻してみましょう。

GitHub DesktopアプリのHistoryからCubeを置いたコミット（バージョン）を選び右クリックします。

表示されたメニューから「Revert changes in commit」をクリックします。
これで新しく「リバートコミット（バージョン）」ができます。
このようにリバートは実はコミットの一種です。

そしてUnityを選択すると画像のようにリロードを勧められます。

リロードすると先ほど置いたはずのCubeが消えています。
このようにUnityプロジェクトの時を戻すことができました。

場合によってはリロードがうまくできず、Unityを閉じてLibraryフォルダを消すなどの対応が必要かもしれません。

またリバートは実は理想的な方法ではありません。
慣れてきたら「git reset」について調べてみてください。

Pullとは？

この記事で扱う最後の操作としてプル（Pull）をご紹介します。
プルはクラウド上のバージョンをローカルに持ってくる操作です。

先ほどご紹介したクローンはローカルに無いリポジトリを新しく持ってくる操作でした。
それに対してプルはすでにローカルにあるリポジトリに、新しいバージョンだけを持ってくる点が違います。

Web上で新しいバージョンをコミットする

まず説明のために「ローカルには無いがクラウド上にだけあるコミット（バージョン）」を作ります。

再びWeb上でリポジトリのトップ画面から、以前の手順で作ったテキストファイルを編集しましょう。

そしてコミットします。
先ほどは説明を省略しましたが、Web上でのコミットは同時にプッシュまで行われるイメージです。

Pullする

ではあなたのローカルPCに戻ってプルしましょう。

GitHub Desktopアプリでの操作は簡単で、上部の「Repository」メニューを開いて「Pull」ボタンをクリックするだけです。

これでWeb上、つまりローカルPC以外で作られたコミット（バージョン）をローカルPC上に取得できました。

この操作は複数台のPCで交互に作業する場合や、後々他の方と共同で制作する際に頻繁に使います。
またそういう場合にはコンフリクト（Conflict）やマージ（Merge）について知っておくと便利です。

注意事項

PrivateとPublic

GitHubは元々プログラマ向けのソースコード共有サービスとして設計されています。
そのためソースコードの公開に適した機能や設定が多くあります。

例えばリポジトリ作成画面ではあらかじめ「Public」が選択されています。

しかしVRChat向けのUnityプロジェクトや3Dモデルは公開したくない場合がほとんどでしょう。

また次のようなデータは大抵の場合公開や再配布が禁止されています。

  • Unity Asset Storeで入手したアセット
  • Boothで入手したモデルデータ

そのため、リポジトリ作成時にはPrivate（あるいは意図してPublic）に設定できているか確認してください。
なおPrivate／Publicは後から切り替えることができます。

ファイルは100MBまで／リポジトリは5GBまで

GitHubにPushできるファイルやリポジトリ自体にはサイズの上限があります。

ファイルは１つあたり100MBまで、リポジトリ自体は5GBまでしか扱うことができません。
適度にリポジトリを分割する、あるいはGit LFSなどの高度な機能を使うことでこの制限を回避できます。

無料アカウントは1人1つまで

GitHub利用規約/B. アカウント利用規約/3. アカウントの要件

1 人の個人または 1 つの法人が複数の無料「アカウント」を保持することはできません

この記事でご紹介した方法を使って、期限なく無償でUnityプロジェクトや他のファイルを管理できます。
しかし無償のGitHubアカウントは１人１つまでと決められていますのでご注意ください。
